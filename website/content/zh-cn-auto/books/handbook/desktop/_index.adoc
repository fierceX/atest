[[desktop-synopsis]]
== 概要

尽管FreeBSD因其性能和稳定性而在服务器领域很受欢迎，但它也非常适合作为日常桌面使用。在FreeBSD ports树中有超过{numports}个应用程序可用，可以轻松构建一个定制的桌面环境，可以运行各种各样的桌面应用程序。本章介绍了如何安装流行的桌面环境以及诸如网络浏览器、办公软件、文档查看器和财务软件等桌面应用程序。

先决条件：

* 阅读本章的读者应该已经了解如何在FreeBSD上安装crossref:x11[x11，X Window System]或crossref:wayland[wayland，Wayland]。
* 在本章中，读者被指示安装官方软件包。请参考crossref:ports[ports-using,使用端口集合]部分，以从端口构建定制软件包。

....

[[kde-configuration]]
==== 配置KDE Plasma

KDE Plasma使用man:dbus-daemon[1]作为消息总线和硬件抽象层。这个应用程序会自动作为KDE Plasma的依赖项安装。

在`/etc/rc.conf`中启用D-BUS服务以在系统启动时启动：

[source, shell]
....
# sysrc dbus_enable="YES"
....

要增加消息大小，请执行以下操作：

[source, shell]
....
sysctl net.local.stream.recvspace=65536
sysctl net.local.stream.sendspace=65536
....

[[kde-start]]
==== 启动KDE Plasma

首选的KDE Plasma显示管理器是package:x11/sddm[]。要安装package:x11/sddm[]，请执行以下操作：

[source, shell]
....
# pkg install sddm
....

在`/etc/rc.conf`中启用SDDM服务以在系统启动时启动：

[source, shell]
....
# sysrc sddm_enable="YES"
....

可以通过运行以下命令在SDDM中设置键盘语言（例如，对于西班牙语）：

[source, shell]
....
# sysrc sddm_lang="es_ES"
....

启动KDE Plasma的第二种方法是手动调用man:startx[1]。为了使其工作，需要在~/.xinitrc中添加以下行：

[source, shell]
....
% echo "exec ck-launch-session startplasma-x11" > ~/.xinitrc
....

....

proc                    /proc           procfs  rw              0       0
....

在`/etc/rc.conf`中启用SDDM服务以在系统启动时启动：

[source, shell]
....
# sysrc sddm_enable="YES"
....

在SDDM中，可以通过运行以下命令来设置键盘语言（例如，对于西班牙语）：

[source, shell]
....
# sysrc sddm_lang="es_ES"
....

启动LXQT的第二种方法是手动调用man:startx[1]。为了使其工作，需要在`~/.xinitrc`中添加以下行：

[source, shell]
....
% echo "exec ck-launch-session startlxqt" > ~/.xinitrc
....
